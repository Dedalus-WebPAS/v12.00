Most Java enabled browsers don't print Java applets along with HTML text and images. Exceptions to this are Internet Explorer version 4 and Hot Java. For other browsers, 2 options are available:

1) If the browser supports Java 1.1 (Navigator 4, IE 4), you can create a signed applet that will be able to access print methods. The applet still won't print the accompanying HTML, but JavaChart graphics are produced with full fidelity. The JavaChart Developers Kit includes a sample showing how to use these print classes and methods.

2) You can use JavaChart to produce GIF versions of your graphics, which can be saved or printed. For most browsers, you must first save the GIF information on your http server, and then send it back to the browser to be saved or printed locally. With some intelligent scripting on your server, it'seven possible to embed the JavaChart GIF image(s) into an HTML page, replacing Java graphics with GIF graphics.

It's also possible to use JavaChart classes as a servlet, producing GIF output for use on any browser, even those without Java support.

When you click the button below this chart, JavaChart will build a GIF stream from the chart and send that stream back to Visual Engineering's server. Next, your browser will display a GIF version of the chart, which is printable.
