About The Applets

JavaChart applet behavior is defined by using the APPLET and PARAM tags. These charts can be embedded into your documents in a static manner, but they're much more interesting if you build them on the fly. You can accomplish this by building a gateway application that writes out the appropriate HTML text with JavaChart APPLET and PARAM tags, or you can use a static chart definition that updates its data dynamically with JavaChart's URL options.

The Applets

The current JavaChart basic applet collection includes the following:

  • lineApp.class
    • Line charts are useful for plotting data that has variability in both X and Y observations. The line chart applet does not stack data; it represents each data set strictly as defined. Datasets can be defined with Y data only, or with both X and Y data. Multiple datasets can included variable numbers of observations. The line charts applet also creates an option legend that describes the datasets with a color and string.

      Line charts that require non-numeric labels for the X-axis are created using the labelLineApp.class applet, which recognizes parameters for user-defined X-axis labels.

      Line charts with discontinuous data can use disLineApp.class applet, which represents "holes" in data definitions with missing line segments.

      Regression plots use a different applet, regressApp.class, but share all other characteristics with Line Charts. The last plot on the Line Chart page shows an automatic regression.

      Line Charts can also use dates or time stamps for X data if they are constructed using the dateLineApp.class applet. This applet uses a DateAxis class for the X axis, which automatically creates a time or date specific axis scale.

      Click here to see some sample line charts.

  • finComboApp.class
    • Creates a combination financial information chart. This chart can contain HLOC data, stick chart data, or line graph data, all combined into a single graph. This series of applets also includes hlocApp.class, which creates HLOC charts, and stickApp.class, which creates stick charts. These charts are all able to process a wide variety of date strings to match whatever logging mechanisms or database output you can create.

      Click here to see some sample finance charts.

  • areaApp.class
    • Creates a stacked area chart. This looks similar to a line graph, but the y values for each data set are "stacked" to create an area that represents a combined total. This applet would be useful for representing something like the combined total income from several different product lines over a particular time period.

      You can also create an area chart that uses a DateAxis by choosing the dateAreaApp.class applet. This applet considers X data values to be time stamps, and will automatically create a time/date scale for the X axis.

      Click here to see some sample area charts.

  • barApp.class
    • Creates a horizontal bar chart. Bar Charts can have labels on each bar as well as on each axis location. Charts that have multiple datasets cluster the datasets into groups. Bar Charts can have a legend that describes each dataset with a description and an icon. Click here to see some sample bar charts.
  • columnApp.class
    • This is the same as a bar chart, but with vertical bars instead of horizontal bars. Click here to see some sample column charts.
  • stackBarApp.class
    • This applet is the same as a bar chart, but data values from multiple data sets are stacked, rather than clustered. Click here to see some sample stacked bar charts.
  • stackColumnApp.class
    • Similar to a column chart, but data values from multiple data sets are stacked, instead of clustered. Click here to see some sample stacked column charts.
  • indBarApp.class
    • This chart builds horizontal bars, but gives each bar a unique color. This type of chart is useful for emphasizing that the compared categories are unique. This applet will also create a legend that describes each data element with a color and an icon. Click here to see some sample individual bar charts.
  • indColumnApp.class
    • This is the same as indBarApp.class, except that the bars are vertical instead of horizontal. Click here to see some sample individual column charts.
  • hiLoBarApp.class
    • This is also called a floating bar chart or a norm comparison chart. It uses a set of Y values along with difference values to create a series of floating bars. These charts are commonly used to represent high and low values, or deviations, differences, etc. Click here to see some sample hiLoBar charts.
  • pieApp.class
    • Pie charts are most useful for comparing categories of data. The pie chart applet can build a chart with or without slice labels, which can show percentages, descriptions, or values. This applet can also build a legend to describe its contents. Pie charts can also be elliptical, and can be placed off-center. The pie applet also can explode, or highlight, specified pie slices. Click here to see some sample pie charts.
  • speedoApp.class
    • Speedo charts show a single data value on a dial or guage. The scale on a speedo chart is constructed the same way as an axis, with automatic scaling being the default. These charts have only 1 axis, the Y axis, and use only 1 point out of an array of data series: the first point of the first dataset. Speedo charts can be full, semi, or quarter circles, with major and minor ticks. They can also have line, arrow, pointer, or sweep needles. Speedo charts scale to the size and shape of the plotarea definition (plotAreaTop, plotAreaRight, etc.) Click here to see a bunch of sample speedometers.

Active Applets

JavaChart also includes applets that can be manipulated by user input. These use standard JavaChart functions, combined with Java user interface controls to provide various kinds of interactive chart. These include the following:

  • scrollingLine.class
    • This applet permits you to scroll through a large amount of data, using a standard AWT scrollbar. Click here for a sample.
  • zoomLine.class
    • This applet lets you click and drag to zoom in on a selected range of data. Use mouse button 2 or 3 to reset the chart. Click here for a sample.
  • spinningPie.class
    • Users can click and drag to rotate a PieChart using this applet. Click here for a sample.
  • hyperBar.class
    • This applet lets you define a series of bars with active hyperlinks to other URLs. This is implemented as both a BarChart and a ColumnChart.

Using the Applets

JavaChart applets use conventional HTML tags for Java applets. The "APPLET" tag names one of the classes listed above. This tag is followed by any number of "PARAM" tags, which specify the look and behavior of the applet on the page. Probably the easiest way to get started with a JavaChart applet is to grab the HTML source from one of the samples linked to this page, and then make whatever changes you require.

Each applet requires some of the files in the JavaChart package, the applet class itself, and a class file called "ChartAppShell.class". To use the software on your own server, you will need to make sure your server has at least the following files:

  • javachart/applet/appletName.class
  • javachart/applet/ChartAppShell.class
  • javachart/chart/*

General Applet Parameters

Because of JavaChart's modular construction, some of the applet parameters are shared by all the applets, and some are specific to particular kinds of charts. The following parameters are shared by the entire applet collection, and are related to general charting features. This table and those that follow use these conventions for describing the kind of argument each parameter requires:

  • anything
    • anything goes here. Really. These parameters are usually for turning something on or off, so for clarity's sake you might want to use something like PARAM name=legendOn value="true"
  • color
    • this field expects a color name, or a hexadecimal color definition (in RGB). Valid colors in these applets include black, white, gray, darkGray, lightGray, red, pink, orange, yellow, green, magenta, cyan, and blue. A valid hex definition for white is "ffffff"
  • font
    • font parameters include information for the font name, the font size, and the font style. Any valid Java font works here, but we start with a default of TimesRoman 12pt in most cases to ensure that the font is available. A font parameter looks like this: param name=xAxisLabelFont value="TimesRoman,18,2". This instructs JavaChart to use 18 point Times Roman italic fonts for this chart's X axis labels
  • integer
    • this parameter requires an integer (any number without a decimal point)
  • double
    • this parameter requires an number, generally a number between 0 and 1. The applets use these to describe percentages, e.g. .5 = 50%
  • list
    • a few parameters require a list of arguments. These include things like axis labels and data values, but they also include lists of axis options.

Dataset Specification

Datasets can vary a great deal from chart to chart. Some kinds of charts only use Y values, others use both X and Y values. Some charts use a color to represent an entire dataset (bar charts, line graphs), and some use a color for each data value (pie charts, individual bar charts). In general, every chart must define a list of numbers for Y data. Charts that don't define this data will use an internally generated fake dataset, which is pretty useless. For additional ways to define data, see the table on URL datasets.

URL Dataset Parameters

These applets include a mechanism to retrieve data from a valid URL. Current Java restrictions in Navigator generally require this URL to use the same host the applet came from. You can use this mechanism to keep your chart definitions and Web pages unchanged while modifying data files as frequently as you wish. You can also specify a network retrieval interval that causes the JavaChart applets to re-read data from the specified URL, updating the chart as necessary.
 

URL Dataset Parameters (available datasets 0 through 19)
dataset0xURL URL URL for a file of comma separated X values for dataset 0
dataset0yURL URL URL for a file of comma separated list of Y values for dataset 0
URLDataBlock URL URL for a file of comma separated Y values. Each line in this file is assumed to be a unique dataset.
URLXYDataRows URL URL for a file of comma separated XY pairs arranged into rows. Each row represents a single dataset, with values arranged as x1, y1, x2, y2, x3, y3... 
URLXYDataColumns URL URL for a file of comma separated XY pairs arranged into columns. This file must contain a single value as the first line of the file, specifying the number of observations (rows) in the file. The first two columns are dataset 1, the second two columns are dataset 2, etc.
networkInterval integer Number of seconds to wait before re-reading URL datasets
customDatasetHandler string information passed to a user-defined dataset handling routine "void getMyDatasets(String str)" Note: in the case of DateAxis, this is the URL for a data file that contains columnar Date information
